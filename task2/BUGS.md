# BUGS — task2

### BUG-1: Не получается создать объявление

- Эндпоинт: `POST /api/1/item`
- Шаги воспроизведения:
	1. Отправить POST-запрос с данными (пример: `sellerID: 123456`, `name`, `price`)
- Ожидаемый результат: Статус 200, объявление создано
- Фактический результат: Статус 400 Bad Request
- Приоритет: High

---

### BUG-2: Возвращает все объявления вместо нужных

- Эндпоинт: `GET /api/1/:sellerID/item`
- Шаги воспроизведения:
	1. Отправить `GET /api/1/999999/item` (несуществующий продавец)
- Ожидаемый результат: Пустой массив `[]` или ошибка 404
- Фактический результат: Возвращается множество объявлений всех продавцов
- Приоритет: Critical

---

### BUG-3: Неправильный статус для несуществующего объявления

- Эндпоинт: `GET /api/1/item/:id`
- Шаги воспроизведения:
	1. Отправить `GET /api/1/item/nonexistent123`
- Ожидаемый результат: Статус 404 Not Found
- Фактический результат: Статус 400 Bad Request
- Приоритет: Medium

---

### BUG-4: Пустой ID возвращает неправильный статус

- Эндпоинт: `GET /api/1/item/:id`
- Шаги воспроизведения:
	1. Отправить `GET /api/1/item/` (пустой id)
- Ожидаемый результат: Статус 400 Bad Request (некорректный формат ID)
- Фактический результат: Статус 404 Not Found
- Приоритет: Low

---
