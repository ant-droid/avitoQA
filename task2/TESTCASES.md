# Тест-кейсы для API микросервиса Avito

## Общая информация
- Host: https://qa-internship.avito.com
- Уникальный sellerID должен быть в диапазоне 111111-999999

## 1. POST /api/1/item - Создать объявление

### Позитивные тест-кейсы:

**TC_001: Создание объявления с корректными данными**
- **Предусловие:** Нет
- **Шаги:**
  1. Отправить POST запрос на /api/1/item
  2. Передать корректные данные в теле запроса:
     - sellerID: 123456 (в диапазоне 111111-999999)
     - name: "Тестовое объявление"
     - price: 1000
     - statistics: {likes: 0, viewCount: 0, contacts: 0}
- **Ожидаемый результат:**
  - HTTP статус: 200 OK
  - В ответе присутствуют поля: id (string), sellerId (integer), name (string), price (integer), statistics (object), createdAt (string)
  - Все поля заполнены корректными значениями

**TC_002: Создание нескольких объявлений для одного продавца**
- **Предусловие:** TC_001 пройден успешно
- **Шаги:**
  1. Создать первое объявление для sellerID: 123456
  2. Создать второе объявление для того же sellerID: 123456
- **Ожидаемый результат:**
  - Оба объявления созданы успешно (статус 200)
  - У объявлений разные id
  - Одни и те же sellerId

### Негативные тест-кейсы:

**TC_003: Создание объявления без обязательного поля sellerID**
- **Шаги:**
  1. Отправить POST запрос без поля sellerID
- **Ожидаемый результат:**
  - HTTP статус: 400 Bad Request
  - В ответе есть сообщение об ошибке

**TC_004: Создание объявления с некорректным sellerID (меньше минимального)**
- **Шаги:**
  1. Отправить POST запрос с sellerID: 100000
- **Ожидаемый результат:**
  - HTTP статус: 400 Bad Request
  - В ответе есть сообщение об ошибке валидации

**TC_005: Создание объявления с некорректным sellerID (больше максимального)**
- **Шаги:**
  1. Отправить POST запрос с sellerID: 1000000
- **Ожидаемый результат:**
  - HTTP статус: 400 Bad Request
  - В ответе есть сообщение об ошибке валидации

## 2. GET /api/1/item/:id - Получить объявление по ID

### Позитивные тест-кейсы:

**TC_006: Получение существующего объявления**
- **Предусловие:** TC_001 пройден успешно (есть id объявления)
- **Шаги:**
  1. Отправить GET запрос на /api/1/item/{id} с существующим id
- **Ожидаемый результат:**
  - HTTP статус: 200 OK
  - В ответе данные соответствуют созданному объявлению
  - Присутствуют все обязательные поля

### Негативные тест-кейсы:

**TC_007: Получение несуществующего объявления**
- **Шаги:**
  1. Отправить GET запрос с несуществующим id (например, "nonexistent123")
- **Ожидаемый результат:**
  - HTTP статус: 404 Not Found
  - В ответе есть сообщение об ошибке

**TC_008: Получение объявления с некорректным форматом ID**
- **Шаги:**
  1. Отправить GET запрос с id в некорректном формате (например, пустой строкой)
- **Ожидаемый результат:**
  - HTTP статус: 400 Bad Request
  - В ответе есть сообщение об ошибке валидации

## 3. GET /api/1/:sellerID/item - Получить все объявления по sellerID

### Позитивные тест-кейсы:

**TC_009: Получение всех объявлений для существующего продавца**
- **Предусловие:** TC_001 и TC_002 пройдены успешно (есть несколько объявлений для sellerID: 123456)
- **Шаги:**
  1. Отправить GET запрос на /api/1/123456/item
- **Ожидаемый результат:**
  - HTTP статус: 200 OK
  - В ответе массив из 2 объявлений
  - Все объявления принадлежат sellerId: 123456
  - Присутствуют все обязательные поля для каждого объявления

### Негативные тест-кейсы:

**TC_010: Получение объявлений для несуществующего продавца**
- **Шаги:**
  1. Отправить GET запрос для несуществующего sellerID (например, 999999)
- **Ожидаемый результат:**
  - HTTP статус: 200 OK или 404 Not Found (в зависимости от реализации)
  - Если 200: пустой массив []
  - Если 404: сообщение об ошибке

**TC_011: Получение объявлений с некорректным sellerID**
- **Шаги:**
  1. Отправить GET запрос с sellerID в некорректном формате (строка вместо числа)
- **Ожидаемый результат:**
  - HTTP статус: 400 Bad Request
  - В ответе есть сообщение об ошибке валидации

- **Ожидаемый результат:**
